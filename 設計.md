ロボコンの大型LEDに映像を表示するためのシステム

# データ形式

## シーン
0 ～ 4個の映像ストリームを持つ

### 持っているデータ

- シーンID
- シーンname
- シーンに属する映像ストリームIDのリスト

## 映像ストリーム

### 持っているデータ
- 映像ストリームID
- 映像ストリームname
- 映像ストリームURL
- RTSP URL
- 接続状態 (接続中、未接続)

# 画面

## 本番映像
シーンを表示 (2x2のグリッド)
装飾は全くなし
下部に、シーンIDとシーンname、接続状態を表示
WebRTCで映像ストリームを表示

## 全ストリーム
すべての映像ストリームをグリッドで表示
(確認用)
装飾は全くなし
下部に、接続状態を表示
WebRTCで映像ストリームを表示

## 設定、管理画面
- どのシーンを表示するか
- シーンごとの映像ストリームの管理

# 技術
- フロントエンド: React + Vite
- バックエンド: Python + FastAPI
- データベース: SQLite

## バックエンド
- 映像は、RTSPで受け取る
- MediaMTXを使用して、WebRTCで配信する
- 設定画面からは、RTSPのURLを登録、バックエンド側でMediaMTXのAPIを叩いて、WebRTCで配信するようにする

それぞれの画面同士は、socket.ioで通信する。
変更がリアルタイムで反映されるようにする。
