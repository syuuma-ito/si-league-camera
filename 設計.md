ロボコンの大型LEDに映像を表示するためのシステム

# データ形式

## システム全体の状態
- active_scene_id: 現在メイン画面(`/main`)で表示すべきシーンのID
- 各映像ストリームの接続状態（接続試行中、接続中、切断）カメラ～MediaMTX間の接続状態
- 各映像ストリームの接続状態（接続試行中、接続中、切断）MediaMTX～フロントエンド間の接続状態

死活監視について、カメラ～MediaMTX間は MediaMTX API 、MediaMTX～フロントエンド間は、フロントエンドから、情報を取るようにする


## シーン
0 ～ 4個の映像ストリームを持つ

### 持っているデータ

- シーンID(uuid)
- シーンname(str)
- 映像ストリームの配置リスト
  - stream_id: 映像ストリームID
  - position: `top-left`, `top-right`, `bottom-left`, `bottom-right` のいずれか

## 映像ストリーム

### 持っているデータ
- 映像ストリームID(uuid)
- 映像ストリームname(str)
- MediaMTXパス(str): MediaMTX上で管理するためのパス名（例: `cam1`）

# 画面

完全にクローズドなネットワーク上なので、セキュリティは考慮しない
装飾は最低限

## 本番映像
"/main"

2×2のグリッド
数がすくない場合も2×2のグリッドで表示

アクティブなシーンを表示
装飾は全くなし
下部に、現在のシーンIDとシーンname、それぞれの接続状態を表示
WebRTCで映像ストリームを表示

切断時には、プレースホルダー画像を表示する

## 全ストリーム
"/streams"

すべての映像ストリームをグリッドで表示
(確認用)
装飾は全くなし
下部に、それぞれの接続状態を表示
WebRTCで映像ストリームを表示

## 設定、管理画面
"/settings"

- どのシーンを表示するか（active_scene_id の変更）
- シーンごとの映像ストリームの管理（配置の変更）
- 映像ストリームの登録・編集・削除

使いやすく、過剰な装飾は避ける

# 技術
- フロントエンド: React + Vite
- バックエンド: Python + FastAPI

## バックエンド
- 映像は、RTSPで受け取る
- MediaMTXを使用して、WebRTCで配信する
- SQLiteと、SQLModelで、シーンや映像ストリームの情報を管理する
- データなどは、Pydanticなど、型安全にする


## 通信
REST API: ストリーム・シーンの登録・編集・削除といった CRUD 操作。
WebSocket: active_scene_id の切り替えの即時反映、各ストリームの接続状態（死活監視）のリアルタイムなブロードキャスト。

設定変更やアクティブシーンの変更がリアルタイムで反映されるようにする。
すべて型安全にする
